预备工作

在开始之前你要知道的东西

动态网站？静态网站？

  • 动态网站是指不仅有前台页面，可以展示给用户看的web页面，也有后台处理各种数据的服务端。通常用于编写服务端的语言有Java, php等。

  • 静态网站是指仅仅有给用户查看的web页面，不能根据不同用户做出的不同操作进行不同的反应的就是静态网站。

Git 是什么

  • Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

  • Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

  • Git 与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，不必服务器端软件支持。

Github Pages是什么

  • Github Pages 是 Github 为用户提供的一个免费的域名，在这个域名里面，你可以放置一个静态的网站。通常 Github Pages 被大多数人用来做个人Blog，当然你也可以用来存储一些东西，方便你来查阅。本篇文章会按照常规的方法教你建一个 个人Blog，如果你想做的是别的东西而不是个人Blog，相信你看完这篇文章也会知道怎么做了。

Hexo 是什么

  • 如果你有丰富的前端功底，你也可以选择手撸代码，自己写html, css, js来展示你的前端页面。但是，即使是那些经验丰富的老前端，要做一个个人Blog，也不可能会手磨前端，肯定会用一个便于管理和控制自己写的几百个文章的工具——我们要介绍的Hexo。

  • Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

**_config.yml**

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds 模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source 资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes 主题 文件夹。Hexo 会根据主题来生成静态页面。

配置站点

认识 *_config.yml*

*_config.yml* 是一个很重要的文件，之后我们会需要常常提起或修改它。现在我们通过它来更改我们的站点设置。

请注意，并不是所有的东西都要修改。下示 *_comfig.yml* 我全部打了注释的原因是让你们更好的理解hexo的原理。一般只需要更改 # Site 下的 title, description, author, language, timezone 以及 # Deployment 下面的东西就够了。

修改完 _config.yml 之后，在 hexo 的 根目录 通过 git bash 或 cmd 输入 hexo g。在做这一步之前确保你已经在前文中进行过了hexo init

常用的hexo命令：

命令 作用
hexo g 生成文件
hexo d 部署生成的文件
hexo clean 清楚生成的文件
hexo s 在本地启动服务器 (多用于预览效果)

获得 personal access token

做这件事之前要做另一件事。

从 2021年8月13日 开始，Github 不在支持通过输入账号密码的形式push代码，需要使用 personal access token 个人访问令牌。其实 github 很早之前就提示过的，不过当时没留意。那么自己去 github 上自己生成个 personal access token 就可以了，所以以后就是通过账号和 personal access token 来验证用户了，而不需要输入密码了（感觉更安全了）。

那么这个 personal access token 怎么弄到呢？

进入 Github ，点击头像，再点击 Settings。进入 Settings 之后再点击 Developer settings。再点击 Personal access token

点击生成令牌

为你的令牌添加一个描述性名称

为你的令牌添加一个到期期限(建议拉到最大)

给自己用的东西当然是全选啦~ 如果你觉得不安全也可以只选 repo

最后再单击生成令牌

请注意，令牌生成之后无法再次在Github中查看，一定要在重新打开或关闭该页面之前将令牌保存下来。如果因为种种原因遗失令牌，请删除令牌之后再按照上述步骤重新生成。
